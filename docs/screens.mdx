このページでは、HeatupNavi の主要画面を「どこを見て・何を判断するか」の観点で整理します。

:::note スクリーンショット準備中
各セクションに、後からスクリーンショットを貼れるように枠を用意しています。
:::

## ホーム：今日の沸き上げ

アプリを開いたときに最初に見る画面です。

### 1) 「おすすめ時間帯」カード

- 表示されるのは、**沸き上げをするのにおすすめの時間帯**です。
- 右上には、状況に応じて **湯切れリスク（低/中/高）** のバッジが出ます。
- **湯切れリスクが高い**場合は、追加で **「安全側（湯切れ対策）」** が表示され、
  - より安全な時間帯
  - その場合の追加コスト目安（`+○円目安`）
  が確認できます。

:::tip 「詳しく見る」で根拠まで確認できます
カードをタップすると詳細（おすすめの理由、必要な沸き上げ時間など）が見られます。
:::

:::note スクリーンショット
ここにホームの「おすすめ時間帯」カードのスクリーンショットを追加
:::

### 2) 「昼/夜どっちが得？」カード

- 昼側（太陽マーク）と夜側（月マーク）で、**どちらの時間帯が実質的に得か**を比較します。
- 補足として **「実質単価 = 電気単価 ÷ 効率（COP）」** が表示されます。

:::note スクリーンショット
ここに「昼/夜どっちが得？」カードのスクリーンショットを追加
:::

### 3) 「推定節約額」カード

- **基準運用** と **おすすめ運用** の推定差を「円」で表示します。
- 月/年は「直近の記録日の平均」を日数換算した概算です（記録が少ないほどブレます）。

:::caution 「算出に必要な入力が不足しています」と出るとき
単価（昼/夜）や使用湯量など、節約額の計算に必要な入力が不足している状態です。
[設定](/docs/settings) と [前日の使用湯量](/docs/usage) を確認してください。
:::

### 4) 「データ状態」カード

- **天気**：最終更新日時
- **使用湯量**：
  - 前日分の実績が入力されている
  - 直近平均で推定している
  - 未入力（標準量で計算）
  のいずれかが表示されます。

### 5) 右上の操作（更新 / 履歴 / 沸き上げ停止）

- **更新**：天気や計算結果を再取得します。
- **履歴**：節約額などの推移を棒グラフで確認します。
- **沸き上げ停止**：旅行・出張など不在のとき、
  - その日の沸き上げ計算と通知を停止します
  - **翌日には自動的に再開**します

## おすすめ詳細（「詳しく見る」）

ホームの「おすすめ時間帯」をタップすると表示されます。

ここでは主に、次が確認できます。

- **必要な沸き上げ時間**（どれくらい運転が必要か）
- **おすすめの時間帯（セグメント）**（いつ動かすのがよいか）
- **おすすめの理由**（天気・単価・生活制約などの影響）

## 設定

ホーム下部の **「設定」** から開きます。

- 通知（毎朝7:00）
- 電気単価（円/kWh：昼/夜）
- 生活制約（使える時間/避けたい時間、最短連続運転、最大分割数）
- プロ設定（タンク容量・貯湯温度・給水温・保温ロス）
- ソーラーパネル（売電単価、パネル容量、方角、影）
- 自宅位置の再登録
- データ（履歴CSV書き出し、iCloud同期）

詳しくは [設定の詳細](/docs/settings) を参照してください。

---

## 前日の使用湯量入力

ホーム下部の **「前日の使用湯量」** から開きます。

- 最近の平均（直近7回）が表示されることがあります
- 「少なめ/ふつう/多め」プリセットから選ぶ
- `+10/-10` で微調整
- `+50/+100/+150` で「1日だけ増やす」
- 必要ならキーボードで数値を直接入力

---

## 履歴

ホーム右上の **「履歴」** から開きます。

- 月/年の切り替え
- 棒グラフをタップすると詳細が表示
- 合計節約、平均/日、記録日数の要約

履歴は [設定 → データ](/docs/settings#データ) から CSV で書き出しできます。
